# 業務要件一覧

| id | category | requirement | priority | description |
| --- | --- | --- | --- | --- |
| BR-001 | ユーザー・アカウント管理 | ロール別アカウント管理プロセス | 高 | SmashHubでは、Admin、Facility Owner、Userの3種類のロールを区別してアカウントを管理する。各ロールは業務上の責任範囲に応じて、参照・編集・管理できる情報や実施可能な業務が異なる。詳細な権限マトリクスは未定義のため、ロール別の業務権限範囲は別途確認する※要確認。 |
| BR-002 | ユーザー・アカウント管理 | Userアカウント登録プロセス | 高 | プレイヤーは、SmashHubの利用開始時にUserアカウントを登録し、施設予約、チーム参加、対戦相手募集、決済などの利用者向け業務を行える状態になる。登録時に必要な項目、本人確認の要否、登録承認の有無は未定義のため確認する※要確認。 |
| BR-003 | ユーザー・アカウント管理 | Facility Ownerアカウント登録プロセス | 高 | 施設オーナーは、施設管理業務を行うためにFacility Ownerアカウントを登録する。Facility Ownerは施設開設申請、施設情報管理、予約確認、売上確認などの業務を担当する。アカウント登録と施設開設申請の前後関係、本人・事業者確認の要否は未定義のため確認する※要確認。 |
| BR-004 | ユーザー・アカウント管理 | Adminアカウント管理プロセス | 高 | Adminは、プラットフォーム全体の管理者として、UserおよびFacility Ownerのアカウント情報や状態を確認し、必要に応じて編集、停止、削除などの管理対応を行う。Adminアカウントの作成者、付与条件、複数管理者の承認フローは未定義である。 |
| BR-005 | ユーザー・アカウント管理 | ユーザープロフィール登録・更新プロセス | 高 | Userは、プロフィール情報としてAvatar、氏名、電話番号、Email、CCCDなどを登録・更新する。これらの情報は、予約、チーム活動、決済、本人識別に利用される想定である※要確認。各項目の必須・任意区分、変更時の確認手続きは未定義である。 |
| BR-006 | ユーザー・アカウント管理 | Facility Ownerプロフィール管理プロセス | 中 | Facility Ownerは、施設運営業務に必要なアカウント情報を管理する。施設情報そのものとは別に、オーナー個人または事業者としての連絡先、受通知先、管理担当者情報を保持する想定である※要確認。管理対象項目と施設情報との境界は未定義である。 |
| BR-007 | ユーザー・アカウント管理 | アカウント情報参照プロセス | 高 | Adminは、全UserおよびFacility Ownerのアカウント一覧および詳細を参照し、状態確認や違反対応などの管理業務に利用する。Facility OwnerおよびUserが他者のアカウント情報をどの範囲まで参照できるかは未定義のため確認する※要確認。 |
| BR-008 | ユーザー・アカウント管理 | アカウント情報編集プロセス | 中 | アカウント保有者は、自身のプロフィール情報を必要に応じて更新する。Adminは、運営上必要な場合にUserまたはFacility Ownerの状態や一部情報を編集できる。Adminによる編集対象項目、本人への通知要否、変更履歴の扱いは未定義である。 |
| BR-009 | ユーザー・アカウント管理 | アカウント状態管理プロセス | 高 | アカウントは、業務上の利用可否を示す状態で管理される。想定される状態には、有効、停止、BAN、削除済みなどが含まれる※要確認。状態遷移の条件、各状態で利用可能な業務範囲、復旧可否は未定義である。 |
| BR-010 | ユーザー・アカウント管理 | ポリシー違反時のBANプロセス | 高 | Adminは、利用規約またはプラットフォームポリシーに違反したUserまたはFacility Ownerに対して、アカウントのBANを行う。違反検知の起点、警告の有無、BAN期間、異議申し立て手続きは未定義のため確認する※要確認。 |
| BR-011 | ユーザー・アカウント管理 | アカウント削除プロセス | 高 | Adminは、ポリシー違反や運営上の必要に応じてアカウントを削除できる。UserまたはFacility Owner本人から退会・削除申請を受け付ける業務が存在するかは未定義である。削除前に予約、チーム、施設、決済、売上など関連業務への影響確認が必要になる想定である※要確認。 |
| BR-012 | ユーザー・アカウント管理 | ロール変更・権限変更プロセス | 中 | アカウントのロールおよび権限は、業務上の役割に応じて管理される。UserがFacility Ownerを兼ねることができるか、同一人物が複数ロールを保持できるか、Admin権限を後から付与・解除できるかは未定義であるため確認する※要確認。 |
| BR-013 | スポーツ種目・レベル管理 | スポーツ種目マスタの登録 | 高 | Adminは、プラットフォーム上で利用可能なスポーツ種目を業務上の管理対象として登録する。登録対象には種目名を含める。種目コード、表示順、公開状態などの管理項目は未定義のため※要確認。 |
| BR-014 | スポーツ種目・レベル管理 | スポーツ種目マスタの変更 | 高 | Adminは、登録済みスポーツ種目の名称や関連情報を必要に応じて変更する。既に施設、ユーザー、予約、チーム、マッチメイキング等で利用されている種目を変更する場合の業務ルールは未定義のため※要確認。 |
| BR-015 | スポーツ種目・レベル管理 | スポーツ種目マスタの削除 | 中 | Adminは、不要となったスポーツ種目を削除対象として扱う。削除時に既存ユーザーの競技レベル、施設対応種目、予約履歴、料金設定などに影響がある場合の扱いは未定義のため※要確認。業務上は、物理削除ではなく利用停止として扱う可能性がある※要確認。 |
| BR-016 | スポーツ種目・レベル管理 | 種目別レベル体系の設定 | 高 | Adminは、各スポーツ種目に対して競技レベルを定義する。レベルには名称、Rank value、説明文を設定する。例として、種目ごとに「初心者」「中級者」などの名称、数値化されたRank value、レベル説明を管理する想定※要確認。 |
| BR-017 | スポーツ種目・レベル管理 | 種目別レベル情報の変更 | 高 | Adminは、登録済みの種目別レベルについて、名称、Rank value、説明文を変更する。既にユーザーが登録しているレベルを変更した場合、そのユーザープロフィールやマッチメイキング条件へ反映する業務タイミングは未定義のため※要確認。 |
| BR-018 | スポーツ種目・レベル管理 | 種目別レベル情報の削除 | 中 | Adminは、不要となった種目別レベルを削除対象として扱う。既存ユーザーに紐づくレベル、チーム募集条件、対戦相手募集条件で利用中の場合の削除可否や代替レベルへの移行手順は未定義のため※要確認。 |
| BR-019 | スポーツ種目・レベル管理 | ユーザーの競技レベル登録 | 高 | Userは、自身のプロフィールにおいて、プレイするスポーツ種目と該当するレベルを登録する。1ユーザーが複数種目を登録できる想定※要確認。登録された競技レベルは、プロフィール表示、チーム活動、対戦相手募集、マッチメイキング時の判断材料として利用される。 |
| BR-020 | スポーツ種目・レベル管理 | ユーザーの競技レベル更新 | 中 | Userは、自身の競技経験や実力変化に応じて、登録済みの種目別レベルを更新する。更新時に過去の参加実績、マッチング履歴、チーム内統計へ影響を与えるかは未定義のため※要確認。 |
| BR-021 | 施設申請・施設情報管理 | 施設開設申請の受付 | 高 | Facility Ownerは、SmashHub上で施設を公開・運営するため、施設開設申請を行う。申請時には、施設名、施設概要、住所、電話番号、対応スポーツ、営業時間、施設画像・動画など、審査に必要な情報を提出する。必須項目の詳細は未定義※要確認。 |
| BR-022 | 施設申請・施設情報管理 | 申請中施設のステータス管理 | 高 | 施設開設申請は、申請後にAdminによる審査対象となり、申請中、承認済み、却下などの業務ステータスで管理される。ステータスの正式名称および遷移条件は未定義※要確認。 |
| BR-023 | 施設申請・施設情報管理 | Adminによる施設申請内容の確認 | 高 | Adminは、Facility Ownerから提出された施設開設申請の内容を確認し、施設情報、所在地、連絡先、対応スポーツ、営業時間、掲載メディアなどがプラットフォーム掲載基準を満たしているかを確認する。審査基準の詳細は未定義※要確認。 |
| BR-024 | 施設申請・施設情報管理 | 施設申請の承認 | 高 | Adminは、申請内容に問題がない場合、施設開設申請を承認する。承認された施設は、Facility Ownerによる運営管理およびUser向けの検索・予約対象として扱われる※要確認。 |
| BR-025 | 施設申請・施設情報管理 | 施設申請の却下 | 高 | Adminは、申請内容に不備または掲載基準違反がある場合、施設開設申請を却下する。却下時にFacility Ownerへ理由を通知するか、再申請を許可するかは未定義※要確認。 |
| BR-026 | 施設申請・施設情報管理 | 施設プロフィールの登録・更新 | 高 | Facility Ownerは、承認済みまたは申請中の施設について、施設名、紹介文、Avatar、施設概要などのプロフィール情報を登録・更新する。更新内容が即時反映されるか、Adminの再審査を必要とするかは未定義※要確認。 |
| BR-027 | 施設申請・施設情報管理 | 施設住所情報の管理 | 高 | Facility Ownerは、施設の所在地情報を登録・更新する。住所はUserが施設を選択・訪問するための基本情報として管理される。地図表示、緯度経度、住所変更時の再審査要否は未定義※要確認。 |
| BR-028 | 施設申請・施設情報管理 | 施設電話番号の管理 | 中 | Facility Ownerは、施設への問い合わせや連絡に使用する電話番号を登録・更新する。電話番号の公開範囲、複数電話番号の登録可否、確認手続きは未定義※要確認。 |
| BR-029 | 施設申請・施設情報管理 | 施設画像・動画ライブラリの管理 | 中 | Facility Ownerは、施設紹介のために画像および動画を登録・更新・削除する。登録対象には、施設Avatar、施設外観、コート、設備紹介などが含まれると想定される※要確認。登録可能なファイル種別、件数、審査要否は未定義。 |
| BR-030 | 施設申請・施設情報管理 | 週間営業時間の設定 | 高 | Facility Ownerは、施設の曜日別営業時間を設定する。営業時間はUserの予約可能時間や施設情報表示の前提として扱われる。祝日、臨時休業、特別営業時間の扱いは未定義※要確認。 |
| BR-031 | 施設申請・施設情報管理 | 対応スポーツの登録・管理 | 高 | Facility Ownerは、施設で利用可能なスポーツ種目を登録・更新する。対応スポーツは、Userの施設検索、予約対象、料金設定の前提情報として扱われる。Adminが管理するスポーツ種目マスタから選択する想定※要確認。 |
| BR-032 | 施設申請・施設情報管理 | スポーツ別の施設基本情報管理 | 中 | Facility Ownerは、対応スポーツごとに平均時間単価や利用可能情報などの基本情報を管理する。詳細な料金設定は別カテゴリのコート・料金管理で扱うが、施設情報として表示する平均価格は本カテゴリで管理対象となる※要確認。 |
| BR-033 | 施設申請・施設情報管理 | 施設情報公開内容の管理 | 中 | 承認済み施設のプロフィール、住所、電話番号、営業時間、対応スポーツ、画像・動画などは、Userが施設を確認するための公開情報として扱われる。公開・非公開をFacility Ownerが項目単位で制御できるかは未定義※要確認。 |
| BR-034 | 施設申請・施設情報管理 | 施設情報変更時の確認プロセス | 中 | Facility Ownerが施設名、住所、対応スポーツ、画像・動画などの重要情報を変更した場合、業務上Admin確認を必要とする可能性がある※要確認。どの項目を重要変更とみなすか、変更前情報を保持するか、承認前に公開するかは未定義。 |
| BR-035 | コート・料金管理 | 施設内コート情報の登録・管理 | 高 | Facility Ownerは、承認済み施設に対して施設内のコートを登録し、コート名、対応スポーツ種目、利用可否、説明情報などを管理する。コートは料金設定および予約枠管理の単位として扱う。コートの詳細属性は未定義のため※要確認。 |
| BR-036 | コート・料金管理 | スポーツ種目別のコート対応管理 | 高 | Facility Ownerは、各コートで利用可能なスポーツ種目を設定する。1つのコートが複数スポーツに対応できるか、または1コート1種目に限定するかは未定義のため※要確認。 |
| BR-037 | コート・料金管理 | 平均時間単価の設定 | 中 | Facility Ownerは、施設またはスポーツ種目単位で平均時間単価を設定する。平均時間単価は施設情報上の目安料金として利用され、実際の予約料金は曜日・時間帯・コート別の詳細料金が優先される想定※要確認。 |
| BR-038 | コート・料金管理 | 曜日別料金の設定 | 高 | Facility Ownerは、スポーツ種目およびコートごとに、曜日別の料金を設定する。平日、週末、祝日をどのように区分するかは未定義のため※要確認。 |
| BR-039 | コート・料金管理 | 時間帯別料金の設定 | 高 | Facility Ownerは、各曜日に対して時間帯ごとの料金を設定する。例として、同一コートでも14:00〜17:00は60,000VND、17:00〜21:00は70,000VNDのように異なる料金を設定できる業務とする。時間帯の最小単位は未定義のため※要確認。 |
| BR-040 | コート・料金管理 | 競技別・コート別・曜日別・時間帯別の動的料金管理 | 高 | Facility Ownerは、スポーツ種目、コート、曜日、時間帯の組み合わせ単位で料金を管理する。予約時には該当する組み合わせの料金が適用される。複数条件に一致した場合の優先順位は未定義のため※要確認。 |
| BR-041 | コート・料金管理 | 料金未設定時の扱い | 中 | 対象のスポーツ種目・コート・曜日・時間帯に個別料金が設定されていない場合、平均時間単価または標準料金を適用する想定とする※要確認。標準料金の定義および料金未設定時に予約不可とするかどうかは未定義。 |
| BR-042 | コート・料金管理 | 料金変更の適用タイミング管理 | 高 | Facility Ownerが料金を変更した場合、変更後に作成される予約に新料金を適用し、既存予約には予約時点の料金を維持する想定※要確認。料金変更の即時適用、将来日指定適用、既存予約への再計算可否は未定義。 |
| BR-043 | コート・料金管理 | 料金設定内容の確認業務 | 中 | Facility Ownerは、登録済みの料金設定を一覧またはカレンダー形式で確認し、スポーツ種目、コート、曜日、時間帯ごとの料金に誤りがないか確認する。確認画面の形式は未定義のため※要確認。 |
| BR-044 | コート・料金管理 | コート利用停止時の料金管理 | 中 | Facility Ownerは、メンテナンスや運用都合によりコートを利用停止にする場合、当該コートの料金設定を保持したまま予約対象から除外する想定※要確認。利用停止期間の指定方法、既存予約がある場合の扱いは未定義。 |
| BR-045 | 予約・スケジュール管理 | 空き枠確認プロセス | 高 | Userまたはチームリーダーは、施設、スポーツ種目、コート、日付、時間帯を指定して予約可能な空き枠を確認できる業務フローとする。空き状況は、オンライン予約およびFacility Ownerが手動登録した外部予約を反映して判定する。検索条件の必須項目や表示単位は未定義※要確認。 |
| BR-046 | 予約・スケジュール管理 | オンライン予約申請プロセス | 高 | Userまたはチームリーダーは、空き枠を選択し、利用するスポーツ種目、コート、利用日時、参加チームまたは参加者情報を指定してオンライン予約を申請する。予約者が個人かチームリーダーに限定されるかはチーム利用時はリーダーのみと想定する※要確認。 |
| BR-047 | 予約・スケジュール管理 | 予約確定プロセス | 高 | オンライン予約申請後、対象枠に他の予約が存在しない場合、予約を確定扱いまたは確定待ち扱いとして管理する。予約確定の条件が決済完了、施設承認、即時確定のいずれであるかは未定義※要確認。 |
| BR-048 | 予約・スケジュール管理 | 予約重複防止ルール | 高 | 同一施設、同一コート、同一日付、重複する時間帯に対して、複数の予約が成立しない業務ルールとする。オンライン予約とFacility Ownerによる外部予約の両方を対象に重複判定を行う。時間帯の重複判定方法、準備・片付け時間の扱いは未定義※要確認。 |
| BR-049 | 予約・スケジュール管理 | 予約ステータス管理 | 高 | 予約は、申請中、確定、変更申請中、キャンセル済み、完了などのステータスで業務上管理する想定とする※要確認。各ステータスの正式名称、遷移条件、誰が変更できるかは未定義※要確認。 |
