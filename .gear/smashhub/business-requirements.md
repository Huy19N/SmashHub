# 業務要件一覧

| id | category | requirement | priority | description |
| --- | --- | --- | --- | --- |
| BR-001 | 顧客管理 | 顧客情報の新規登録プロセス | 高 | 営業担当者が新規取引先を獲得した際、規定の登録項目（会社名、代表者、住所、電話番号、法人番号など）を入力してシステムに新規登録を行う。 ※要確認（詳細な登録ルールや社内審査プロセスの有無は未定義のため、一般的な手続きを想定） |
| BR-002 | 顧客管理 | 顧客情報の更新および履歴管理プロセス | 中 | 既存顧客の社名変更、移転、連絡先変更などが発生した際、担当者が情報を変更し、更新履歴（変更者、変更日時、変更前後の内容）を記録・管理する。 ※要確認（変更時の承認フローの有無や対象項目は未定義のため、一般的な履歴管理を想定） |
| BR-003 | 顧客管理 | 取引先担当者情報の紐付けおよび異動管理業務 | 高 | 1つの顧客（企業）に対して、複数の取引先担当者（部署、役職、氏名、連絡先など）を登録・管理し、担当者の異動や退職時には履歴を保持したままステータスを更新する。 ※要確認（複数担当者の管理方法や異動時の業務運用は未定義のため、一般的な運用を想定） |
| BR-004 | 顧客管理 | 検索条件による顧客情報の照会業務 | 高 | 営業および管理部門のスタッフが、顧客名、業種、担当者、エリア等の任意の条件を組み合わせて検索し、必要な顧客情報を一覧表示・確認できる。 ※要確認（詳細な検索絞り込み条件や部門ごとの閲覧権限ルールは未定義） |
| BR-005 | 顧客管理 | 重複顧客データの防止・クレンジング業務 | 中 | データのサイロ化を防ぐため、新規登録時に既存データとの重複（会社名や法人番号の類似）をチェックし、重複が疑われる場合は名寄せ・統合を行う。 ※要確認（名寄せの具体的な判断基準や運用フローは未定義のため一般的な運用を想定） |
| BR-006 | 案件・進捗管理 | 案件登録・初期起票プロセス | 高 | 営業活動の発生に伴い、営業担当者が対象顧客に紐付く新規案件を登録する業務フロー。登録時は、案件名、想定商材、想定売上金額、受注予定時期を必須定義とする。 ※要確認（具体的な必須項目やビジネスルールは未提供のため、一般的な営業プロセスを想定して定義） |
| BR-007 | 案件・進捗管理 | 進捗ステータス（フェーズ）の更新・管理ルール | 高 | 案件の進捗段階（「アプローチ」「ヒアリング」「提案・見積」「交渉」「成約」「失注」など）を明確に定義し、商談の進展に合わせてタイムリーにステータスを更新する業務プロセス。 ※要確認（具体的なフェーズ定義や遷移のルール・ビジネスルールは未定義） |
| BR-008 | 案件・進捗管理 | 案件担当者および関係者の割り当てプロセス | 中 | 起票された案件に対して、主担当者（プライマリ営業）および副担当者（技術支援・サポート等）をアサインし、チーム内での役割分担を明確化する業務プロセス。 ※要確認（体制変更時の引き継ぎルール等は未定義） |
| BR-009 | 案件・進捗管理 | 商談履歴および活動記録の登録プロセス | 高 | 顧客との商談、架電、メール等の接触が発生した都度、担当者はその活動内容、顧客の反応、次回のアクション、課題点を履歴として記録し、関係者へ共有する業務フロー。 ※要確認（活動記録の登録期限や報告ライン等のビジネスルールは未定義） |
| BR-010 | 案件・進捗管理 | 見積提示・提案時のビジネスフロー | 中 | 顧客に対して正式な見積や提案を行う前に、提示金額や提案内容についてチーム内または上長の確認を得た上で、提示プロセスを進める業務フロー。 ※要確認（金額に応じた決裁権限やワークフローの有無は未定義） |
| BR-011 | 案件・進捗管理 | 失注・中断時における要因分析プロセス | 中 | 進行中案件が「失注」または「中断」となった際、営業担当者がその具体的な要因（価格、競合、納期、機能不足、時期尚早など）を分類・記録し、次回の営業戦略や製品改善に活かすための振り返りプロセス。 ※要確認（失注要因の分類定義は未提供） |
| BR-012 | 案件・進捗管理 | 案件情報の定期的な状況見直しプロセス | 高 | 週次または月次の営業会議等のタイミングで、確度（成約見込）や受注予定時期、見積金額等の案件情報を最新化し、全社的な売上見通しの精度を担保する見直しプロセス。 ※要確認（更新頻度や会議体の詳細は未定義） |
| BR-013 | 案件・進捗管理 | 顧客情報と案件情報のデータ紐付けルール | 高 | 新規案件の登録にあたり、必ず顧客管理（CRM）に登録されている有効な顧客データに紐付けることを必須とし、データの重複や無関係なレコードの作成を防止する業務統制プロセス。 ※要確認（新規顧客における同時登録プロセスの有無は未定義） |
| BR-014 | 売上集計・ダッシュボード | 案件完了に伴う売上データの自動集計業務 | 高 | 案件管理プロセスにおいて、案件ステータスが「受注」または「完了」となったデータを、自動的に売上データとして抽出し集計対象とする。※要確認（売上認識の基準タイミングや売上確定となる案件ステータスが未定義のため、一般的な受注時を想定） |
| BR-015 | 売上集計・ダッシュボード | 部門別の売上データ集約・分類業務 | 高 | 登録された売上データを、営業担当者の所属部門や案件に紐づく部門ごとに自動分類して集約する。※要確認（複数部門で売上を按分するルールや、兼務担当者の売上帰属ルールが未定義のため、単一主所属への紐づけを想定） |
| BR-016 | 売上集計・ダッシュボード | 複数期間（月次・四半期・年次）での売上集計業務 | 高 | 指定した任意の集計期間（月、四半期、年度など）に基づき、売上実績の自動集計を行い、時系列での売上推移を算出する。※要確認（会計年度の開始月や、締日変更などの例外処理ルールが未定義のため、4月開始・月末締めを想定） |
| BR-017 | 売上集計・ダッシュボード | 主要KPI（売上目標達成率）の算出業務 | 中 | 部門ごと、または全社ごとに設定された売上目標値（予算）と売上実績値を突き合わせ、目標達成率や進捗率を自動で算出する。※要確認（目標値の登録・管理プロセスや、KPIの具体的な計算式が未定義のため、一般的な目標達成率［実績/目標］を想定） |
| BR-018 | 売上集計・ダッシュボード | 集計データのダッシュボード可視化（グラフ表示）業務 | 中 | 集計された部門別売上や期間別売上のデータを、棒グラフや折れ線グラフ等の視覚的なレポートとしてダッシュボードへリアルタイムに表示し、経営・管理層の意思決定を迅速化する。※要確認（表示対象とするグラフの種類や推奨レイアウトが未定義のため、一般的な売上推移・構成比グラフを想定） |
| BR-019 | 売上集計・ダッシュボード | 集計対象データのCSV出力業務 | 高 | ダッシュボードや集計画面に表示された売上集計データおよびその明細データを、他システム（会計システム等）への取り込みや二次利用のために、CSV形式でダウンロード可能とする。※要確認（出力フォーマットの仕様や連携先システムが未定義のため、標準的な売上明細レコード形式を想定） |
| BR-020 | 売上集計・ダッシュボード | 過去売上実績データの蓄積・比較照会業務 | 中 | 経年での売上分析や前年同期比の算出を可能とするため、過去複数年にわたる売上実績データを保管し、現在のデータとシームレスに比較照会できるようにする。※要確認（データ保存期間や参照制限に関するルールが未定義のため、過去5年間分を保管対象とすることを想定） |
