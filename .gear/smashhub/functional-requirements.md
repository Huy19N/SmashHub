# 機能要件一覧

| id | functionName | description | relatedRequirement | priority |
| --- | --- | --- | --- | --- |
| FR-001 | 顧客情報登録・編集画面 | 顧客の基本情報（会社名、代表者、住所、電話番号、法人番号等）の登録、更新、詳細確認を行う画面（CRUD - CUD）。※要確認（詳細な入力制限や、外部サービスによる法人番号・住所の自動入力支援機能の要否は未定義のため、手動入力を想定） | BR-001, BR-002 | 高 |
| FR-002 | 顧客変更履歴管理機能 | 顧客情報が更新された際、変更日時、変更担当者、変更前後のデータを自動的に履歴テーブルに記録するバックエンド機能（CRUD - C）。※要確認（変更内容に対するマネージャー承認などの監査ログ要件は未定義のため、一律自動記録を想定） | BR-002 | 中 |
| FR-003 | 取引先担当者紐付け登録機能 | 1つの顧客企業に対して、複数の取引先担当者（部署、役職、氏名、連絡先等）を登録・紐付けし、異動・退職等のステータスを更新する画面（CRUD - CUD）。※要確認（担当者の兼務や、主担当者以外の関連度合いの定義方法は未定義） | BR-003 | 高 |
| FR-004 | 顧客検索・一覧照会画面 | 顧客名、業種、担当者、エリア等の複数条件での検索、および一覧表示・詳細画面への遷移機能（CRUD - R）。※要確認（ログインユーザーの所属部門による顧客データの閲覧権限制限ルールの有無は未定義） | BR-004 | 高 |
| FR-005 | 顧客重複チェック・名寄せ機能 | 新規顧客登録時に、同一の会社名や法人番号が既に登録されていないかを判別して警告を表示する機能。および既存重複データの手動マージ機能。※要確認（名寄せの合算ロジック［どちらの履歴を残すか等］は未定義のため、手動選択による統合を想定） | BR-005 | 中 |
| FR-006 | 案件登録・編集画面 | 顧客に紐付く新規案件（案件名、想定商材、想定売上金額、受注予定時期、進捗ステータス等）を登録、更新する画面（CRUD - CUD）。※要確認（新規顧客登録と同時に案件を起票する「一括作成機能」の要否は未定義） | BR-006, BR-013 | 高 |
| FR-007 | 案件進捗ステータス更新機能 | 「アプローチ」「ヒアリング」「提案・見積」「交渉」「成約」「失注」などの進捗ステータスを更新し、変更日時のログを残す機能（CRUD - U）。※要確認（特定の進捗フェーズをスキップして変更することを禁止する等の制御ルールの有無は未定義） | BR-007 | 高 |
| FR-008 | 案件担当者割り当て機能 | 起票した案件に対して、システムユーザーマスタから主担当者（1名）および副担当者（複数名可）をアサインする画面機能（CRUD - U）。※要確認（主担当者の変更時に旧担当者への通知、および顧客の営業担当マスタとの自動同期の要否は未定義） | BR-008 | 中 |
| FR-009 | 商談履歴・活動記録登録画面 | 案件や顧客に紐付く商談日、接触方法、対応内容、次回予定、課題等を時系列で登録・参照する機能（CRUD - CR）。※要確認（活動履歴に対する変更・削除を制限するセキュリティルールの有無は未定義のため、登録者のみ編集可を想定） | BR-009 | 高 |
| FR-010 | 見積・提案事前確認申請機能 | 見積書や提案書を作成し、顧客に提示する前に上長に対してシステム上で承認申請を行う簡易ワークフロー機能（CRUD - CU）。※要確認（見積金額ごとの承認ルートの変更や、多段階承認の要件は未定義のため、1次上長による単一承認を想定） | BR-010 | 中 |
| FR-011 | 失注・中断要因記録機能 | 案件ステータスを「失注」または「中断」に変更する際、あらかじめ定義された要因カテゴリ（価格、納期、競合等）の選択と詳細理由のテキスト入力を必須化する制御機能（CRUD - U）。※要確認（失注要因の分類マスタ管理画面の要否は未定義） | BR-011 | 中 |
| FR-012 | 案件一括見直し画面 | 営業会議等での利用を想定し、担当案件の一覧から進捗度、見積金額、受注予定時期を直接グリッド上で一括更新できる機能（CRUD - U）。※要確認（一度に更新可能な件数制限などの制限事項は未定義） | BR-012 | 高 |
| FR-013 | 売上データ自動抽出バッチ | 案件ステータスが「成約（受注）」または「完了」となった案件データを検知し、売上集計用テーブルに自動同期する夜間バッチ処理機能（CRUD - C）。※要確認（バッチの稼働周期、実行ログの出力先、障害時のリトライ仕様などの非機能要件は未定義） | BR-014 | 高 |
| FR-014 | 部門別・期間別売上集計処理機能 | 蓄積された売上データを、案件の主担当者が所属する部門、および指定された期間（月次・四半期・年次）に基づき、集計用データベース上で自動計算・分類する機能。※要確認（複数部門への売上按分機能や、月途中の組織変更時の売上按分基準ルールは未定義） | BR-015, BR-016 | 高 |
| FR-015 | 売上目標マスタ管理画面 | 部門別、年度別、月別の売上目標（予算）を設定、登録、編集、削除する画面（CRUD - CRUD）。※要確認（目標データのCSVによる一括インポート機能の要否は未定義） | BR-017 | 中 |
| FR-016 | 目標達成率（KPI）算出ロジック機能 | 指定の期間・部門における「売上目標マスタ」と「売上実績集計値」を突合し、売上目標達成率（実績 / 目標）を自動算出する演算処理（CRUD - R）。※要確認（特定の例外部門を除外する、もしくは特定商材の売上のみを対象にする等の計算除外ルールは未定義） | BR-017 | 中 |
| FR-017 | 売上ダッシュボード画面（グラフ表示） | 集計された部門別売上推移や各部門の目標達成率を、グラフ（棒グラフ、折れ線グラフ、円グラフ等）を用いてポータル画面上にリアルタイム表示する画面。※要確認（ダッシュボード全体の画面配置や、リフレッシュ頻度等の仕様は未定義） | BR-018 | 中 |
| FR-018 | 集計データCSVダウンロード機能 | ダッシュボードおよび売上集計画面に表示されている任意のデータを、CSV形式で一括出力する機能。※要確認（文字コード［UTF-8/Shift-JIS］や他システムとの連携を意識した出力ヘッダー行の固定フォーマット仕様は未定義） | BR-019 | 高 |
| FR-019 | 過去売上データ蓄積・比較照会画面 | 現在から過去5年間に遡る売上実績データをDBに保持し、任意の複数年を対象に売上実績の差異（前年同期比など）を表示・比較する画面（CRUD - R）。※要確認（過年度データの保持上限、および過去データのアーカイブ・パージのバッチ要否は未定義） | BR-020 | 中 |
| FR-020 | システム共通・認証および権限管理機能 | ログイン・ログアウト画面、および管理者によるシステム利用ユーザーの作成・更新・削除・権限設定（管理者、マネージャー、営業一般等）機能（CRUD - CRUD）。※要確認（シングルサインオン［SSO］や多要素認証［MFA］、パスワード変更有効期限などのセキュリティ要件は未定義） | 未定義 ※要確認（業務要件一覧にシステム共通のアクセス制御要件がないため、標準的な認証機能を想定して仮設定） | 高 |
